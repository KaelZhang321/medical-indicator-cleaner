# 数据资产补全方案：大模型生成 + 爬虫验证

## Context

项目代码框架已完备（144 tests passed），但业务数据资产严重不足：
- `standard_dict.csv` 158 条，aliases 覆盖不全
- `reference_range_standard.csv` 仅 3 条模板
- `risk_weight.csv` 仅 3 条模板
- `aggregate_rules.yaml` 仅 2 条规则

需要通过「大模型生成初始数据 + 爬虫验证补充」的方式快速补全。

## 策略：哪些数据用什么方式补

| 数据资产 | 大模型生成 | 爬虫验证/补充 | 说明 |
|----------|-----------|--------------|------|
| standard_dict aliases 扩充 | ✓ 生成常见别名变体 | ✓ 爬医学百科验证 | 模型知道大量同义词，爬虫补漏 |
| reference_range_standard | ✓ 生成通用参考范围 | ✓ 爬权威来源校准 | 参考范围有权威出处，必须爬虫校准 |
| risk_weight | ✓ 生成（基于临床知识） | ✗ 无结构化来源可爬 | 风险权重本质是业务决策，模型给初始值 |
| aggregate_rules | ✓ 生成 | ✗ 无需爬虫 | 规则逻辑简单，人工定义即可 |
| standard_dict 新增指标 | ✓ 补充长尾指标 | ✓ 爬指标百科确认 | 如男性特有项(PSA)、电解质等当前缺失的 |

## 爬虫数据源分析

### 可用公开数据源

| 来源 | URL 模式 | 可获取数据 | 结构化程度 | 合规性 |
|------|----------|-----------|-----------|--------|
| **丁香园检验百科** | dxy.com/lab-test/ | 指标名、缩写、别名、参考范围、临床意义 | 高（表格） | 公开医学科普 |
| **百度百科/医学词条** | baike.baidu.com | 指标别名、临床意义 | 中 | 公开百科 |
| **好大夫在线** | haodf.com | 检查项目名称和别名 | 中 | 公开科普 |
| **中国卫生检验标准** | 各检验学教材 | 权威参考范围 | 高但非在线 | 需人工录入 |

### 推荐爬取策略

**主力源：丁香园检验百科**
- 每个检验项目页面有：中文名、英文名/缩写、别名、标本类型、参考范围（分性别年龄）、临床意义
- 结构化程度最高，最适合自动抽取

**辅助源：百度百科**
- 用于补充丁香园缺失的冷门指标
- 提取「又称」「简称」等别名信息

### Step 1: 大模型生成初始数据 `llm_generator.py`

用 Claude API（或火山方舟 doubao）批量生成，不是手写 prompt 让人复制粘贴，而是脚本自动调 API。

**生成任务 1：aliases 扩充**

对 standard_dict.csv 现有 158 条，每条让模型生成更多别名：

```python
prompt = """
你是医学检验专家。对于以下检验指标，列出所有可能的中文别名、英文缩写变体、常见错别字、口语化叫法。

指标标准名：{standard_name}
英文缩写：{abbreviation}
当前已有别名：{existing_aliases}

请补充更多别名，用分号分隔，不要重复已有的。
只输出别名列表，不要解释。
"""
```

**生成任务 2：reference_range_standard 全量填充**

**生成任务 3：risk_weight 全量填充**

**生成任务 4：补充 standard_dict 缺失指标**

当前缺失的常见指标（模型可以直接给出完整行）：

- 电解质：K、Na、Cl、Ca、P、Mg
- 心肌标志物：CK、CK-MB、LDH、cTnI、cTnT、BNP、NT-proBNP

### Step 2: 爬虫验证和补充

#### `crawler_dxy.py` — 丁香园检验百科

```python
class DxyCrawler:
    """爬取丁香园检验百科的检验项目信息"""
    
    BASE_URL = "https://dxy.com"
    
    def crawl_indicator(self, name: str) -> dict | None:
        """
        搜索指标名 → 进入详情页 → 提取：
        - chinese_name: 中文名
        - english_name: 英文名
        - abbreviation: 缩写
        - aliases: 别名列表
        - specimen: 标本类型
        - reference_ranges: [
            {"gender": "all/male/female", "age": "成人/儿童/...",
             "ref_min": float, "ref_max": float, "unit": str}
          ]
        - clinical_significance: 临床意义摘要
        """
    
    def crawl_all(self, names: list[str], delay: float = 1.0) -> list[dict]:
        """批量爬取，每次间隔 delay 秒（礼貌爬取）"""
```

#### `crawler_baike.py` — 百度百科

```python
class BaikeCrawler:
    """从百度百科提取检验指标别名"""
    
    def crawl_aliases(self, name: str) -> list[str]:
        """
        搜索指标名 → 提取信息框中的：
        - 中文名
        - 外文名
        - 别名/又称
        - 简称
        返回所有找到的别名列表
        """
```

#### 爬虫原则
- **间隔 1-2 秒/请求**，不做并发轰炸
- **设置 User-Agent** 为正常浏览器
- **只爬公开科普页面**，不爬登录后内容
- **结果缓存到本地 JSON**，避免重复请求
- **robots.txt 合规检查**

### Step 3: 合并校验 `merger.py`

```python
class DataMerger:
    """合并大模型生成数据和爬虫数据，校验后写入 CSV"""
    
    def merge_aliases(
        self, 
        current_dict: pd.DataFrame,      # 现有 standard_dict
        llm_aliases: dict[str, list],     # 模型生成的别名
        crawled_aliases: dict[str, list], # 爬虫获取的别名
    ) -> pd.DataFrame:
        """
        合并规则：
        1. 爬虫数据优先（更可靠）
        2. 模型数据补充（覆盖更广）
        3. 去重 + 去空值
        4. 标记来源（llm/crawled/original）便于审核
        """
    
    def merge_reference_ranges(
        self,
        llm_ranges: list[dict],
        crawled_ranges: list[dict],
    ) -> pd.DataFrame:
        """
        合并规则：
        1. 爬虫数据优先（有权威出处）
        2. 模型数据填补空缺
        3. 交叉校验：如果两者差异 > 20%，标记为 needs_review
        """
    
    def validate(self, df: pd.DataFrame, schema: str) -> list[str]:
        """校验数据完整性，返回问题列表"""
```

### Step 4: 一键执行 `run_enrichment.py`

```bash
# 完整流程
python scripts/data_enrichment/run_enrichment.py --all

# 只跑大模型生成
python scripts/data_enrichment/run_enrichment.py --llm-only

# 只跑爬虫
python scripts/data_enrichment/run_enrichment.py --crawl-only

# 只合并（已有生成和爬虫结果）
python scripts/data_enrichment/run_enrichment.py --merge-only
```

流程：
1. LLM 生成 → 保存到 `data/enrichment/llm_*.json`
2. 爬虫抓取 → 保存到 `data/enrichment/crawled_*.json`
3. 合并校验 → 生成待审核文件 `data/enrichment/review_*.csv`
4. 人工审核确认后 → 覆盖写入正式数据文件
