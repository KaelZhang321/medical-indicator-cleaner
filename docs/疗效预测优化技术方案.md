# 疗效预测优化技术方案

> 基于《基于多模态医学检查的数据融合与疗效预测》研究报告，结合项目现有数据能力的落地优化方案

---

## 1. 研究报告核心洞察

### 1.1 与我们系统的关联度分析

| 报告主题 | 我们能做 | 我们不能做 | 原因 |
|----------|---------|-----------|------|
| 结构化临床体检（EHR） | **完全可做** | — | 有完整的 HIS 数据库 |
| 实验室化验标志物 | **完全可做** | — | 1970 条标准化指标 |
| 纵向时序分析 | **完全可做** | — | 有多次体检历史 |
| 全身炎症指标(NLR/LDH) | **可计算** | — | 有血常规和生化数据 |
| 影像组学/病理组学 | — | 不可做 | 无 DICOM/WSI 数据 |
| 基因组学/多组学 | — | 不可做 | 无测序数据 |
| 数字孪生 | — | 当前不做 | 需要实时数据流 |

### 1.2 可落地的关键方法论

1. **多特征时序注意力** — 报告提到 MFMAM 模型处理不规则体检记录，预测准确率 >94%。我们有多次体检时序数据
2. **复合炎症指标计算** — NLR（中性粒细胞/淋巴细胞比值）、PLR（血小板/淋巴细胞比值）等衍生指标
3. **共病指数（Charlson）** — 基于诊断历史计算复合风险评分
4. **Delta 特征** — 指标在时间轴上的变化率、加速度、趋势斜率
5. **SHAP 可解释性** — 解释每个特征对预测结果的贡献
6. **风险分层与干预建议** — 将预测结果转化为分层的健康管理策略

## 2. 优化方案设计

### 2.1 现有疗效预测的问题

当前实现（`api/routers/analysis.py` `get_features`）：
- 只看最新一次和上一次体检的简单对比
- 变化率计算过于粗糙（只有 (latest - prev) / prev）
- 缺少时序趋势分析（斜率、加速度、波动性）
- 缺少衍生指标（NLR、LDL/HDL 比值等）
- 缺少综合健康风险评分
- 缺少可解释性（为什么评估为恶化/改善）
- 没有预测能力，只有描述能力

### 2.2 优化后的功能矩阵

| 功能 | 当前 | 优化后 |
|------|------|--------|
| 变化率 | 单次 (latest vs prev) | **多次时序线性回归斜率** |
| 趋势 | 上升/下降/稳定 | **加速恶化/减速恶化/拐点改善/持续改善/波动/稳定** |
| 衍生指标 | 无 | **NLR/PLR/LDL-HDL比/AST-ALT比/eGFR等** |
| 风险评分 | 简单异常计数 | **加权综合评分（0-100）+ 系统级风险** |
| 预测 | 无 | **基于趋势的 6 个月风险预测** |
| 可解释性 | 无 | **每个指标的贡献度排名 + 文字解读** |
| 系统评估 | 简单整体趋势 | **按系统分组评估（心血管/代谢/肝肾/血液/免疫）** |

### 3.2 时序趋势分析引擎

```python
class TimeSeriesTrendAnalyzer:
    """分析多次体检的指标时序趋势"""
    
    def analyze(self, values: list[tuple[str, float]]) -> dict:
        """
        输入: [(date, value), ...] 按时间排序
        输出:
        {
            "slope": float,           # 线性回归斜率（年化）
            "slope_direction": str,   # 上升/下降/稳定
            "acceleration": float,    # 斜率变化率（二阶导数）
            "trend_type": str,        # 加速恶化/减速恶化/拐点改善/持续改善/波动/稳定
            "volatility": float,      # 波动系数 (CV)
            "latest_vs_avg": float,   # 最新值 vs 历史均值的偏差
            "consecutive_abnormal": int, # 连续异常次数
            "predicted_6m": float,    # 基于趋势的6个月预测值
            "confidence": float,      # 预测置信度
        }
        """
```

**趋势类型判定规则**：

| 趋势类型 | 条件 | 临床含义 |
|----------|------|----------|
| 加速恶化 | slope 偏离正常 + acceleration > 0 | 需紧急干预 |
| 减速恶化 | slope 偏离正常 + acceleration < 0 | 当前干预初见成效 |
| 拐点改善 | 最近 2 次 slope 反转 | 治疗方案有效 |
| 持续改善 | slope 朝正常方向 + 连续 ≥3 次 | 可考虑减少复查频率 |
| 波动不定 | volatility > 阈值 | 控制不稳定，需调整方案 |
| 稳定 | |slope| < 阈值 + volatility 低 | 维持现状 |

## 4. 新增模块设计

```
src/
├── derived_indicators.py      # 衍生指标计算引擎
├── trend_analyzer.py          # 时序趋势分析引擎
├── health_assessment.py       # 综合健康评估（整合四象限+趋势+衍生指标）
```

### 4.1 `derived_indicators.py`

- `DerivedIndicatorEngine` 类
- 从基础指标计算 NLR/PLR/LDL-HDL比/AST-ALT比/白球比
- 配置化规则（YAML 或 dict），支持扩展
- 输入：单次体检的标准化 DataFrame
- 输出：追加衍生指标行的 DataFrame

### 4.2 `trend_analyzer.py`

- `TimeSeriesTrendAnalyzer` 类
- 对每个指标做多次体检的线性回归
- 计算斜率、加速度、波动性、连续异常次数
- 6 个月趋势外推预测
- 趋势类型判定

### 4.3 `health_assessment.py`

- `HealthAssessmentEngine` 类
- 整合 QuadrantAnalyzer + TrendAnalyzer + DerivedIndicatorEngine
- 按 BODY_SYSTEMS 分组评估
- 生成综合健康报告（JSON）
- 风险排名和贡献度计算

## 7. 与研究报告的对应关系

| 报告方法论 | 我们的实现 |
|-----------|-----------|
| 多特征时序注意力(MFMAM) | 时序线性回归 + 趋势分类（简化版，无需训练） |
| 常规血液生化+全身炎症(NLR/LDH) | DerivedIndicatorEngine 计算 NLR/PLR 等 |
| 共病指数(Charlson) | 系统级风险评估（按7个系统分组） |
| Delta 影像组学 | Delta 特征（变化率+加速度+趋势斜率） |
| SHAP 可解释性 | 贡献度排名 + 文字解读（规则驱动，非模型驱动） |
| 风险分层与干预建议 | 系统级 status + recommendations |
| 中期融合架构 | 衍生指标 = 跨指标融合（比值类指标本质是特征交叉） |
