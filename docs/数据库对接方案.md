### 2.2 两种结果表结构

| 字段 | hyb (化验室) | ybb/erb/nkb/wkb/fkb/usb (其他科室) |
|------|-------------|-----------------------------------|
| StudyID | ✓ | ✓ |
| SFXMDM (大项) | ✓ | ✗ |
| XXDM (细项) | ✓ | ✓ |

| 单位 | ItemUnit | ✗ (从 jcmxx 取) |

| 日期 | CheckDate | StudyDate |

### 2.3 关键发现

1. **jcmxx 有 1956 条细项**，远比当前 standard_dict 的 158 条丰富 — 这就是最好的「标准字典」来源
2. **jcmxx 中的 XXMC 也有★前缀和括号问题**（如 `★白细胞计数(WBC)`），和 API JSON 数据一样的脏数据模式
3. **jcmxx 没有分性别参考范围数据**（MDefValueUp/Down 全为空），参考范围只在 hyb.DefValue 中
4. **hyb 独有 SFXMDM**，其他科室表没有大项编码，需要通过 XXDM → jcmxx.LBDM 推断分类
5. **Flag 分布合理**：正常 1240万 / 偏高 72万 / 偏低 28万

### 3.3 db_dict_sync.py — 字典同步（最重要）

**核心思路**：用 jcmxx 的 1956 条细项 + sfxm 的 2197 条大项，直接生成/更新本地 standard_dict.csv 和 major_item_dict.csv。

```python
class DictSyncer:
    """从生产数据库同步字典到本地 CSV"""
    
    def sync_standard_dict(self) -> pd.DataFrame:
        """
        从 jcmxx 同步细项字典:
        1. 读取全量 jcmxx
        2. L1 规则清洗 XXMC（去★、提取缩写、统一括号）
        3. 用 EXXMC 作为 abbreviation
        4. 用 LBDM 映射 category
        5. 用 Unit 作为 common_unit
        6. 生成 code = XXDM
        7. 输出为 standard_dict.csv 格式
        """
    
    def sync_major_item_dict(self) -> pd.DataFrame:
        """
        从 sfxm 同步大项字典:
        1. 读取全量 sfxm
        2. 用 SFXMMC 生成 standard_name
        3. 用 KSBM 关联 ksbm 得到科室
        4. 输出为 major_item_dict.csv 格式
        """
    
    def sync_reference_ranges(self) -> pd.DataFrame:
        """
        从 hyb 中聚合参考范围:
        1. 按 XXDM 分组，取 DefValue 最常见的值
        2. 解析为 ref_min, ref_max
        3. 输出为 reference_range_standard.csv
        """
```

### 3.4 db_data_source.py — 数据查询

```python
class DBDataSource:
    """从数据库查询体检数据，输出与 P0Preprocessor 相同格式的 DataFrame"""
    
    def query_by_study_id(self, study_id: str) -> pd.DataFrame:
        """查询单人单次体检数据"""
    
    def query_by_patient(self, sfzh: str) -> list[pd.DataFrame]:
        """查询某人所有历史体检（纵向对比用）"""
    
    def query_by_date_range(self, start: str, end: str) -> pd.DataFrame:
        """按日期范围批量查询（批量清洗用）"""
    
    def _build_hyb_query(self) -> str:
        """
        构建化验室查询 SQL:
        SELECT j.ID as study_id, j.JCRQ as exam_time, j.XM, j.XB,
               x.XMMC as package_name,
               s.SFXMMC as major_item_name, s.SFXMDM as major_item_code,
               m.XXMC as item_name, m.EXXMC as item_name_en,
               h.ItemResult as result_value_raw,
               h.ItemUnit as unit_raw,
               h.DefValue as reference_range_raw,
               h.Flag as abnormal_flag
        FROM ods_tj_hyb h
        JOIN ods_tj_jcxx j ON j.ID = h.StudyID
        LEFT JOIN ods_tj_jcmxx m ON h.XXDM = m.XXDM
        LEFT JOIN ods_tj_sfxm s ON h.SFXMDM = s.SFXMDM
        LEFT JOIN ods_tj_xmzh x ON j.ZHXMDM = x.XMDM
        """
    
    def _build_dept_query(self, table: str) -> str:
        """
        构建其他科室查询 SQL (ybb/erb/nkb/...):
        SELECT j.ID as study_id, j.JCRQ as exam_time,
               m.XXMC as item_name, m.EXXMC as item_name_en,
               d.CValue as result_value_raw,
               m.Unit as unit_raw
        FROM {table} d
        JOIN ods_tj_jcxx j ON j.ID = d.StudyID
        LEFT JOIN ods_tj_jcmxx m ON d.XXDM = m.XXDM
        """
```

## 5. 最大价值：用 jcmxx 重建标准字典

jcmxx 有 1956 条细项，每条都有：
- **XXDM**：编码（天然唯一标识）
- **XXMC**：中文名（需 L1 清洗）
- **EXXMC**：英文名/缩写
- **Unit**：单位
- **LBDM**：类别编码

这比我们手工维护的 158 条 standard_dict 覆盖面大 **12 倍**，而且是生产系统的权威数据。同步后 L1 命中率预计能达到 **90%+**。
