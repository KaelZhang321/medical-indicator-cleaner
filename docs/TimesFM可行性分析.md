# TimesFM 用于疗效预测的可行性分析

> 对 Google TimesFM (https://github.com/google-research/timesfm) 在医疗体检指标预测场景的适用性评估

---

## 1. TimesFM 概述

- **架构**：Decoder-only Transformer，200M 参数，20 层，1280 隐藏维度
- **输入**：32 个连续时间点为一个 patch（类似 LLM 的 token）
- **预训练**：1000 亿真实时间点（Google Trends、Wikipedia、天气、交通等）
- **能力**：零样本预测 + 分位数预测（10th-90th）+ LoRA 微调
- **发表**：ICML 2024，GIFT-Eval 排行榜零样本第一

## 2. 核心结论：不适合当前场景

**三重不匹配导致 TimesFM 不适合我们的数据：**

| 维度 | TimesFM 需要 | 我们的数据 | 匹配度 |
|------|-------------|-----------|--------|
| **数据密度** | 32+ 点才能填满一个 patch | 3-27 点/指标 | 不匹配 |
| **时间间隔** | 固定间隔（日/周/月） | 不规则（月到年） | 不匹配 |
| **序列类型** | 单变量，不考虑指标间关系 | 需要跨指标关联（NLR、LDL/HDL比） | 不匹配 |

### 为什么不匹配

1. **数据太稀疏**：一个患者某个指标只有 3-27 个数据点，而 TimesFM 一个 patch 需要 32 点。69%-91% 的 patch 都是填充值，模型本质上在猜测
2. **时间不规则**：强制重采样为月频后，90%+ 的数据是插值出来的，模型在预测合成数据
3. **单变量限制**：无法建模 ALT 和 AST 的协同变化，或血糖和 HbA1c 的关联

## 3. 与当前方案对比

| 维度 | 当前线性回归 | TimesFM |
|------|------------|---------|
| 最少数据点 | **2 个** | 32 个（实际有效） |
| 不规则间隔 | **原生支持**（转换为天数差） | 需要重采样预处理 |
| 可解释性 | **完全透明**（斜率/加速度/趋势类型） | 黑盒（只有数值输出） |
| 非线性模式 | 不能捕捉 | 可以捕捉（但数据太少无法激活） |
| 置信度 | 启发式公式 | **分位数预测（优势）** |
| 速度 | 微秒级 | 模型加载 2-5 秒 + 推理毫秒级 |
| 依赖 | 零（纯 Python 数学） | PyTorch + 400MB 模型 |
| 临床分类 | **6 种趋势类型 + 系统评估** | 无（只有原始预测值） |

## 4. 推荐方案

### 方案 A（推荐）：增强当前 TrendAnalyzer

**不替换，而是增强现有系统，投入产出比最高。**

| 增强项 | 内容 | 工作量 |
|--------|------|--------|
| 贝叶斯线性回归 | 用 `scipy` 计算正规化置信区间，替代启发式 | 1 天 |
| Bootstrap 重采样 | 对残差做 bootstrap 得到预测区间 | 0.5 天 |
| 分段线性/LOESS | 10+ 数据点时用非线性拟合 | 1 天 |
| 异常检测 | 基于残差的离群点检测 | 0.5 天 |

### 方案 B（可选探索）：TimesFM 作为补充不确定性量化器

仅用于 15+ 数据点的患者，利用其分位数预测能力改善置信度估计：

```python
# 仅对数据量充足的指标使用 TimesFM
if len(series) >= 15:
    # 重采样为月频
    monthly = resample_to_monthly(series, method='locf')
    # TimesFM 分位数预测
    quantiles = timesfm.forecast(monthly, quantiles=[0.1, 0.5, 0.9])
    # 用分位数宽度作为不确定性度量
    uncertainty = quantiles[0.9] - quantiles[0.1]
```

### 方案 C（不推荐当前采用）：不规则时序专用模型

如果未来数据量增长，可考虑专为不规则时序设计的模型：

| 模型 | 特点 | 适用性 |
|------|------|--------|
| **mTAN** (Multi-Time Attention Network) | 专门处理不规则采样 | 高 |
| **GRU-D** | 可训练的缺失数据衰减 | 高 |
| **Neural ODE / Latent ODE** | 连续时间模型 | 高 |
| **Chronos-2** (Amazon) | 另一个时序基础模型 | 中 |

## 5. 如果一定要用 TimesFM 的集成路径

```
原始数据 (3-27 个不规则时间点)
    │
    ▼ 重采样为月频 (LOCF 插值)
    │
    ▼ TimesFM forecast (分位数输出)
    │
    ▼ 后处理层 (映射回趋势类型/风险等级)
    │
    ▼ 与现有 TrendAnalyzer 结果融合
    │
    ▼ 保留现有 TrendAnalyzer 作为 fallback (< 10 个点)
```

硬件需求：
- 模型大小：200M 参数 (~400MB)
- 最低：CPU 推理可用
- 推荐：4GB+ GPU
- 与 FastAPI 兼容，启动时加载一次

## 6. 最终建议

**当前阶段不采用 TimesFM。**

理由：
1. 数据特征（稀疏+不规则）与模型设计（密集+等间隔）根本不匹配
2. 当前线性回归+系统评估已经很好地服务了业务需求
3. 投入产出比极低：需要大量预处理+后处理，效果可能不如简单方法

**正确的投资方向：**
- 优先增强现有 TrendAnalyzer（贝叶斯置信区间、分段拟合）
- 继续积累更多患者多次体检数据
- 当单个指标平均数据点 > 30 个时，再重新评估基础模型方案
