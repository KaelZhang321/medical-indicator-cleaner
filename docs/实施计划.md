# HIS 体检指标名称标准化系统 — 实施计划

## 项目结构

```
medical-indicator-cleaner/
├── config/
│   └── settings.yaml              # 全局配置（模型路径、阈值等）
├── data/
│   ├── standard_dict.csv           # 标准指标字典
│   ├── alias_dict.csv              # 别名词库（人工确认的映射）
│   ├── input/                      # 放原始脏数据
│   │   └── .gitkeep
│   ├── output/                     # 清洗结果输出
│   │   └── .gitkeep
│   └── faiss_index/                # FAISS 向量索引
│       └── .gitkeep
├── docs/
│   ├── 方案设计.md                  # 方案设计文档
│   └── 实施计划.md                  # 本文件
├── src/
│   ├── __init__.py
│   ├── pipeline.py                 # 主流程编排：L1 → L2 → L4
│   ├── l1_rule_cleaner.py          # L1 规则清洗层
│   ├── l2_embedding_matcher.py     # L2 向量召回层
│   ├── l3_reranker.py              # L3 精排判定层（二期预留）
│   ├── l4_review.py                # L4 输出分层 & 审核导出
│   ├── dict_manager.py             # 标准字典 & 别名管理
│   └── utils.py                    # 工具函数
├── scripts/
│   ├── build_index.py              # 构建标准名向量索引
│   ├── run_clean.py                # 一键执行清洗
│   └── review_feedback.py          # 人工审核结果回灌
├── tests/
│   ├── test_l1_cleaner.py          # L1 规则清洗单元测试
│   └── test_pipeline.py            # 端到端测试
├── .gitignore
├── requirements.txt
└── README.md
```

## 一期实施步骤

### Step 1: 项目骨架 + 配置

- [x] 创建项目目录结构
- [ ] 编写 `config/settings.yaml`（模型路径、阈值、文件路径）
- [ ] 编写 `requirements.txt`
- [ ] 编写 `.gitignore`

### Step 2: 标准字典

- [ ] 编写 `data/standard_dict.csv` — 预置 200+ 常见体检指标
  - 血常规（20+ 项）：WBC、RBC、HGB、PLT、MCV、MCH、MCHC...
  - 肝功能（15+ 项）：ALT、AST、GGT、ALP、TBIL、DBIL、ALB...
  - 肾功能（10+ 项）：BUN、Cr、UA、Cys-C...
  - 血脂（6+ 项）：TC、TG、HDL-C、LDL-C、ApoA1、ApoB...
  - 血糖（5+ 项）：FPG、2hPG、HbA1c、GSP、INS...
  - 尿常规（15+ 项）：pH、PRO、GLU、BLD、WBC、NIT...
  - 甲状腺功能（7+ 项）：TSH、FT3、FT4、T3、T4、TgAb、TPOAb...
  - 肿瘤标志物（10+ 项）：AFP、CEA、CA125、CA199、PSA...
  - 其他常见项
- [ ] 编写 `data/alias_dict.csv` — 初始为空，后续通过审核回灌填充
- [ ] 编写 `src/dict_manager.py` — 字典加载、别名索引构建

### Step 3: L1 规则清洗层

- [ ] 编写 `src/l1_rule_cleaner.py` — 8 条链式规则
- [ ] 编写 `tests/test_l1_cleaner.py` — 覆盖各类脏数据场景
- [ ] 验证：确保常见脏数据能正确清洗

### Step 4: L2 向量召回层

- [ ] 编写 `src/l2_embedding_matcher.py`
  - 模型加载（text2vec-base-chinese-sentence）
  - 编码函数
  - FAISS 索引构建与查询
- [ ] 编写 `scripts/build_index.py` — 标准名 + 别名向量索引构建脚本
- [ ] 验证：对几个测试用例确认 Top-5 召回合理

### Step 5: L4 输出分层

- [ ] 编写 `src/l4_review.py`
  - 按阈值三档分流
  - 生成 `auto_mapped.csv`、`need_review.csv`、`manual_required.csv`

### Step 6: Pipeline 串联

- [ ] 编写 `src/pipeline.py` — L1 → L2 → L4 流程编排
- [ ] 编写 `scripts/run_clean.py` — CLI 入口
- [ ] 编写 `src/utils.py` — 通用工具函数

### Step 7: 端到端测试

- [ ] 准备模拟脏数据 `data/input/sample_dirty.csv`
- [ ] 运行完整 pipeline
- [ ] 检查三档输出是否合理
- [ ] 编写 `tests/test_pipeline.py`

### Step 8: 反馈回灌

- [ ] 编写 `scripts/review_feedback.py`
  - 读取人工审核后的 CSV
  - 将确认映射追加到 `alias_dict.csv`
  - 去重处理

### Step 9: README

- [ ] 编写 `README.md` — 使用说明、快速开始、配置说明

## 二期规划（不在本次实施范围）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| L3 精排层 | Cross-Encoder rerank 或大模型 API | P1 |

## 验证标准

1. L1 规则清洗覆盖率 > 50%（精确匹配到标准名的比例）
2. L2 召回 Top-5 命中率 > 90%（正确标准名在 Top-5 中）
3. 自动归一（≥0.95）准确率 > 95%
4. 反馈回灌后重跑，L1 命中率明显提升
