# HIS 体检指标名称标准化系统 — 方案设计

## 1. 背景

HIS 系统中体检指标名称存在大量不规范写法，导致无法自动归类对比展示。例如：

| 问题类型 | 示例 |
|----------|------|
| 多余符号 | `血脂.` vs `血脂` |
| 全半角混用 | `血红蛋白（HGB）` vs `血红蛋白(HGB)` |
| 同义词 | `谷丙转氨酶` vs `ALT` vs `丙氨酸氨基转移酶` |
| 缩写差异 | `WBC` vs `白细胞计数` vs `白细胞` |
| 多余空格 | `血 糖` vs `血糖` |
| 大小写不一致 | `hba1c` vs `HbA1c` |
| 名称混入单位 | `血糖(mmol/L)` vs `血糖` |

数据规模：10,000+ 条不同的脏指标名称。

## 2. 设计原则

- **不是"大模型清洗"，而是归一化系统** — BERT 适合做相似匹配和候选召回，不适合单独硬做全量标准化
- **规则优先，模型兜底** — 规则能解决 50-70% 问题，模型处理剩余的语义匹配
- **必须保留人工兜底** — 医疗数据标准化不追求 100% 自动化
- **闭环反馈** — 人工审核结果回灌别名库，越用越准

## 3. 整体架构

```
原始脏数据
    │
    ▼
┌─────────────────────────────┐
│  L1 规则清洗层               │  正则、全半角、标点、空格、单位分离
│  + 别名词库精确匹配          │  命中 → confidence=1.0 直接输出
└─────────────┬───────────────┘
              │ 未命中
              ▼
┌─────────────────────────────┐
│  L2 向量召回层               │  Sentence-BERT 编码
│  FAISS Top-5 检索           │  返回候选 + 余弦相似度
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│  L3 精排判定层（二期）        │  Cross-Encoder / 大模型 API
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│  L4 输出分层                 │  ≥0.95 自动归一
│                             │  0.80~0.95 待审核
│                             │  <0.80 人工处理
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│  L5 反馈学习层（持续）        │  审核结果 → 回灌别名库
│                             │  下次 L1 直接命中
└─────────────────────────────┘
```

## 4. 标准指标字典

每个标准指标包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `code` | 内部唯一编码 | BL001 |
| `standard_name` | 标准名称 | 总胆固醇 |
| `aliases` | 别名列表（`;` 分隔） | TC;CHOL;胆固醇 |
| `category` | 所属大类 | 血脂 |
| `unit` | 常见单位 | mmol/L |

初始字典覆盖常见体检项：血常规、肝功、肾功、血脂、血糖、尿常规等。

## 5. L1 规则清洗层

链式规则，顺序执行：

1. **strip** — 去前后空格、`\t\n`
2. **全半角统一** — 全角字符转半角（`（）` → `()`、`：` → `:`）
3. **去无意义标点** — 去掉末尾的 `.。,，、-_/`
4. **统一括号** — `【】` → `[]`、`《》` 去掉
5. **提取单位** — 正则分离指标名和单位，如 `甘油三脂mg/dl` → 名称 `甘油三脂` + 单位 `mg/dl`
6. **去内部多余空格** — `血 糖` → `血糖`
7. **英文统一大写** — 用于后续匹配
8. **别名词库精确查找** — 命中则直接返回标准名（confidence=1.0）

## 6. L2 向量召回层

- **模型**：`shibing624/text2vec-base-chinese-sentence`（中文 Sentence-BERT，768 维，离线可用）
- **索引**：FAISS IndexFlatIP（归一化后内积 = 余弦相似度）
- **流程**：
  1. 构建时：标准名 + 所有别名 → 编码 → 存入 FAISS
  2. 查询时：清洗后名称 → 编码 → 检索 Top-5 → 返回候选 + 分数

### 为什么不用 BERT 分类

- 标准项会不断新增，分类模型要频繁重训
- 样本不平衡严重，长尾指标难学
- 短文本容易误判
- **检索匹配比纯分类更稳健**

## 7. 输出分层策略

| 置信度 | 处理方式 | 说明 |
|--------|----------|------|
| ≥ 0.95 | 自动归一 | 直接写入映射表 |
| 0.80 ~ 0.95 | 待人工审核 | 给出 Top-3 候选供选择 |
| < 0.80 | 人工处理 | 可能是新指标，需新增标准项 |

输出字段：`原始名称, 清洗后名称, 匹配标准名, 置信度, 候选Top3, 来源层级(L1/L2)`

## 8. 反馈回灌机制

```
人工审核 CSV → 确认正确映射 → 追加到 alias_dict.csv → 下次 L1 直接命中
```

形成闭环：**运行 → 审核 → 回灌 → 再运行**，命中率持续提升。

## 9. 二期规划

| 能力 | 说明 |
|------|------|
| L3 精排层 | Cross-Encoder rerank 或大模型 API 对 Top-K 精排 |
| 上下文特征 | 加入科室、套餐名、同批次指标辅助判断 |
| 主动学习 | 低置信度样本优先标注，迭代优化模型 |
| OCR 错别字库 | 针对扫描件 OCR 常见错误的专项修正 |
