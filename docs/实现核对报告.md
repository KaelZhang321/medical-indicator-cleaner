## 技术方案核对

### P0 数据预处理

- 支持 HIS JSON 展开为扁平 DataFrame。
- 支持按配置白名单保留 `HY/YB/ER`。
- 支持按 `study_id + dept_code + item_code + item_name` 去重。
- 支持 `resultValue` 中数值、文本、单位、修饰符、判断词、参考范围拆分。
- 支持 `referenceRange` 结构化解析为 `ref_min/ref_max/ref_text/ref_conditions`。
- 支持异常状态补全为 `abnormal_flag_source/is_abnormal/abnormal_direction`。
- 支持单位标准化，输出统一 `unit`。
- 真实样本展开 504 行，过滤后 142 行。

### L1 规则清洗

- 支持去前后空白、`★` 前缀、全半角统一、括号英文缩写提取、尾部标点清理、中文内部空格压缩。
- 支持 `majorItemName` 去 `H-` 前缀。
- 支持名称混入单位的尾部括号清理，如 `血糖(mmol/L)` → `血糖`。
- 支持字典/别名/缩写精确命中并输出 `confidence=1.0`。

### L2 向量召回

- 支持通过 SentenceTransformer 编码标准名、缩写和 aliases。
- 支持 FAISS `IndexFlatIP` 检索。
- 支持同一 `standard_code` 去重并保留最高分候选。
- 支持索引保存到 `faiss.index` 和 `metadata.pkl`，并支持重新加载。
- 单元测试使用可注入 fake encoder，避免测试依赖网络和模型下载。

### L4 输出分层

- 支持 `confidence >= 0.95` 自动归一。
- 支持 `0.80 <= confidence < 0.95` 待审核。
- 支持 `< 0.80` 人工处理。
- 支持导出 `auto_mapped.csv`、`need_review.csv`、`manual_required.csv`、`stats_report.txt`。
- CSV 使用 `utf-8-sig`，便于 Excel 打开中文。
- 统计包含总数、三档数量、L1 命中率、L2 命中率。

### 反馈回灌

- 支持读取人工确认 CSV。
- 仅处理 `confirmed=1`。
- 通过 `DictManager.add_alias()` 写回，保留去重逻辑。
- 已验证新增别名后可立即 lookup 命中。

### AI 复核

- 支持在 L2 之后对 `need_review/manual_required` 记录做二次审核。
- 默认配置模型为 `doubao-seed-2-0-pro-260215`。
- 仅当 `ai_review.enabled=true` 且环境变量 `ARK_API_KEY` 存在时启用。
- AI 明确判断时返回 `auto_map` 并自动归一。
- AI 不确定时返回 `human_review`，最终继续进入人工审核。

### 数据贯通与批量化

- Pipeline 已改为 DataFrame 驱动，不再只传 `item_name`。
- 最终输出保留 P0 原始上下文、解析结果、标准化结果、字典补充字段。
- L1 使用 `clean_batch()` 批量清洗。
- L2 在可用时优先使用 `search_batch()` 批量召回。

## 残留限制

- 真实 Hugging Face 模型索引构建受当前运行环境影响：提权后可开始下载/加载 `shibing624/text2vec-base-chinese-sentence`，但当前环境未稳定产出 `data/faiss_index/faiss.index`。L2 的核心构建、检索、保存和加载逻辑已通过 deterministic fake encoder 覆盖。
- `data/standard_dict.csv` 是高覆盖初版，后续生产使用需要根据人工审核持续扩充 aliases。
- `sample_dirty.csv` 中 `血脂.` 当前仍是人工桶，因为字典中没有“血脂”作为具体标准项；这是合理输出，不属于规则失败。
