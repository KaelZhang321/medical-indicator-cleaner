# 四象限分析技术方案

> 基于《医学体检数据四象限分析》研究报告，结合项目现有数据资产和代码框架的落地实施方案

---

## 1. 目标

将体检标准化后的结构化数据，通过四象限模型进行风险分层可视化，为客户提供：
- 单次体检的全景风险定位（哪些指标紧急、哪些关注、哪些正常）
- 多次体检的风险迁移轨迹（指标是在恶化还是改善）
- 个性化的健康管理建议（不同象限不同策略）

## 2. 四象限模型选型

### 2.1 研究报告中的四种模型

| 模型 | X 轴 | Y 轴 | 适用场景 |
|------|------|------|----------|
| 临床风险×干预可控性 | 临床风险严重度 | 干预可控性(含SDOH) | 慢病人群管理 |
| National Council | 生理健康风险 | 行为/精神健康风险 | 双重诊断 |
| Covey 紧急/重要 | 紧急度 | 重要度 | 医疗团队任务管理 |
| 4QM 学习模型 | 知识掌握度 | 决策能力 | 患者教育 |

### 2.2 我们的选择：指标偏离度 × 临床风险权重

**原因**：我们的数据是体检生化指标，没有 SDOH（社会决定因素）和行为健康数据。最适合的是基于客观检验数据的量化模型。

**坐标定义**：
- **X 轴：指标偏离度（Deviation）** — 检验值偏离参考范围的程度，反映"当前异常严重程度"
- **Y 轴：临床风险权重（Risk Weight）** — 该指标异常时对健康的潜在危害程度，反映"异常的医学重要性"

## 3. 核心算法设计

### 3.1 偏离度计算（现有 `risk_analyzer.calc_deviation`）

当前实现：

```python
def calc_deviation(value, ref_min, ref_max):
    mid = (ref_min + ref_max) / 2
    half_range = (ref_max - ref_min) / 2
    return (value - mid) / half_range  # [-1, 1] 为正常范围
```

**问题**：
1. 对称偏离假设不适用于所有指标（如血糖：偏高比偏低危险得多）
2. 没有区分偏高和偏低的临床意义差异
3. 没有封顶机制，极端值会导致坐标轴失衡

**优化方案 — 非对称加权偏离度**：

```python
def calc_deviation_v2(
    value: float,
    ref_min: float,
    ref_max: float,
    high_weight: float = 1.0,  # 偏高的权重系数
    low_weight: float = 1.0,   # 偏低的权重系数
    cap: float = 3.0,          # 偏离度封顶值
) -> tuple[float, str]:
    """
    返回: (偏离度, 方向)
    - 偏离度: 0 = 参考范围中心, 1.0 = 刚好在参考范围边界
      >1.0 = 超出参考范围, 负数 = 偏低
    - 方向: 'normal' | 'high' | 'low'
    """
    mid = (ref_min + ref_max) / 2
    half_range = (ref_max - ref_min) / 2
    if half_range == 0:
        return (0.0, 'normal')
    
    raw = (value - mid) / half_range
    
    if raw > 0:
        deviation = raw * high_weight
        direction = 'high' if deviation > 1.0 else 'normal'
    else:
        deviation = raw * low_weight
        direction = 'low' if deviation < -1.0 else 'normal'
    
    # 封顶防止极端值
    deviation = max(-cap, min(cap, deviation))
    return (deviation, direction)
```

**常见指标的非对称权重配置（数据资产）**：

```csv

030101,0.8,1.5,白细胞偏低比偏高更危险(免疫缺陷)

```

### 3.2 风险权重体系（现有 `data/risk_weight.csv`）

**优化方案 — 三级权重体系**：

```
最终权重 = 分类基础权重 × 指标调整系数 × 上下文修正系数
```

### 3.3 四象限分类规则（现有 `risk_analyzer.classify_quadrant`）

当前实现：硬编码 `risk_weight >= 0.7 && deviation >= 1`。

**优化方案 — 可配置阈值 + 细分6区**：

```
              高风险权重 (≥0.7)
    ┌────────────┬────────────┐
    │  Q2 重点   │  Q1 紧急   │
    │  关注      │  处理      │
    ├────────────┼────────────┤ 中风险权重 (0.4-0.7)
    │  Q4 常规   │  Q3 轻度   │
    │  观察      │  异常      │
    ├────────────┼────────────┤ 低风险权重 (<0.4)
    │  Q6 健康   │  Q5 生理   │
    │  基座      │  波动      │
    └────────────┴────────────┘
    正常 (|dev|<1)  异常 (|dev|≥1)
```

| 象限 | 偏离度 | 风险权重 | 临床含义 | 建议行动 |
|------|--------|----------|----------|----------|
| **Q1 紧急处理** | ≥1.0 | ≥0.7 | 高危指标严重异常 | 立即就医，专科转诊 |
| **Q2 重点关注** | <1.0 | ≥0.7 | 高危指标尚在范围内 | 密切监测，缩短复查周期 |
| **Q3 轻度异常** | ≥1.0 | 0.4~0.7 | 中危指标超标 | 生活方式干预，定期复查 |
| **Q4 常规观察** | <1.0 | 0.4~0.7 | 中危指标正常 | 年度常规体检 |
| **Q5 生理波动** | ≥1.0 | <0.4 | 低危指标轻微异常 | 无需担心，注意饮食 |
| **Q6 健康基座** | <1.0 | <0.4 | 低危指标正常 | 保持现状 |

### 3.4 综合风险评分（新增）

将四象限分析汇总为一个 0-100 分的健康评分：

```python
def calc_health_score(indicators: list[dict]) -> int:
    """
    评分规则:
    - 基础分 100
    - Q1 每项 -8 分（紧急处理的异常严重扣分）
    - Q2 每项 -3 分（高危但正常的适度扣分）
    - Q3 每项 -5 分（异常但中危）
    - Q4 不扣分
    - Q5 每项 -1 分（低危异常几乎不扣分）
    - Q6 不扣分
    - 最低 0 分
    """
    score = 100
    weights = {'紧急处理': -8, '重点关注': -3, '轻度异常': -5, 
               '常规观察': 0, '生理波动': -1, '健康基座': 0}
    for ind in indicators:
        score += weights.get(ind['quadrant'], 0)
    return max(0, min(100, score))
```

**健康评分等级**：

| 分数 | 等级 | 颜色 | 说明 |
|------|------|------|------|
| 90-100 | 优秀 | 绿色 | 各项指标基本正常 |
| 75-89 | 良好 | 蓝色 | 少量轻微异常 |
| 60-74 | 一般 | 黄色 | 存在需关注的异常 |
| 40-59 | 较差 | 橙色 | 多项指标异常 |
| 0-39 | 危险 | 红色 | 需立即就医 |

## 9. 合规注意事项

基于研究报告第五章的合规分析：

1. **系统定位**：辅助参考工具，不是诊断工具。所有建议文案加"仅供参考，请遵医嘱"
2. **数据安全**：数据库密码和 API Key 走环境变量，不入仓库
3. **隐私保护**：API 返回脱敏身份证号（`1329****0922`），日志不记录完整个人信息
4. **免责声明**：前端页面底部固定显示"本系统分析结果仅供健康管理参考，不构成医疗诊断建议"
